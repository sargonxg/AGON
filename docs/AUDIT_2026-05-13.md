## Architecture-level red flags

### Single-vendor risk (Gemini-only)

**Risk.** All LLM extraction goes through Vertex Gemini. If Gemini API has an outage or a model regression, AGON's perception pipeline degrades or stops.

**Mitigations in plan:**
- **MockLlmBackend** for tests — every CI run independent of vendor.
- **Routing config** (PROMPT 06) — easy to swap in another `LlmBackend` impl.
- **Trait-isolated** — adding a Claude/OpenAI backend later is bounded work, not a rewrite.
- **Schema-constrained** — Vertex's `responseMimeType=application/json` + `responseSchema` is feature-complete; we are not depending on vendor-specific extensions beyond that.
- **Pinned model IDs** — `gemini-2.5-flash-002` etc., recorded in provenance. A bad model rev is detectable and revertible.

**Accept.** Single-vendor is the right MVP choice for cost and operational simplicity. Revisit when first design partner signs.

### Long-context Gemini as adjudication, not primary

**Decision (locked).** Per-document AGON perception remains primary. Gemini 1M context is used only when (a) a primitive's calibrated confidence drops below 0.7 → Pro adjudication, (b) cross-document fusion in DIALECTICA. Never as the first-pass extractor over an entire case folder.

### Evidence-span fidelity

**Risk.** If `aco-text` normalization drops a character we couldn't track, downstream `verify_span` will fail and an entire primitive is rejected. A high reject rate is worse than a low-confidence extraction.

**Mitigation.**
- **Quad-form** spans carry both canonical and raw offsets — recoverable.
- **3-mode verification** (Exact / Normalized / Fuzzy) — graceful degradation.
- **Drift detection** — `normalization_version` semver pinned in every span; bumping it requires a corpus replay.

### Calibration debt

**Risk.** PROMPT 10 lands calibration. Until then every confidence is raw — likely overconfident on LLM outputs and underconfident on deterministic detectors. UI MUST not show raw probabilities to end users before PROMPT 10.

**Mitigation.** Workbench already only shows the friction matrix, not raw per-primitive confidence. Document this constraint in `docs/ETHICS.md` (PROMPT 15).
